## 10 Universal Themes

These principles appear across ALL successful remote-first companies:

### 1. Async-First (Not Async-Only)
> Default to async. Sync requires justification.

**Evidence:** GitLab, Basecamp, Doist, Zapier all default to async but maintain sync options for sensitive/complex situations.

### 2. Written-First Communication
> If it's not written down, it doesn't exist.

**Evidence:** All companies emphasize writing as primary communication method. P2, Twist, Basecamp posts over Slack.

### 3. Public by Default (Internally)
> Share information broadly within the org.

**Evidence:** GitLab's transparency, Buffer's radical openness, PostHog's public handbook.

### 4. Results Over Activity
> Judge by output, not hours or presence.

**Evidence:** No time tracking (Basecamp, GitLab), no surveillance, flexible hours everywhere.

### 5. Tool Purpose Clarity
> Each tool has ONE primary purpose.

**Evidence:** Zapier's tool clarity guide, Doist's Twist vs. other channels distinction.

### 6. Timezone as Feature
> Design for global, not despite it.

**Evidence:** Doist's 10+ timezone coverage, Automattic's 24/7 operations, rotating meeting times.

### 7. Handbook-First Documentation
> Document before implementing.

**Evidence:** GitLab's 2000+ page handbook, Help Scout's "Guide to the Galaxy," Basecamp's public handbook.

### 8. Meeting-Minimal Culture
> Clear criteria for when sync is needed.

**Evidence:** Basecamp's anti-meeting stance, GitLab's "no agenda, no meeting," Doist's rare syncs.

### 9. Trust and Autonomy
> Adults treated as adults.

**Evidence:** No surveillance across all companies, flexible hours, results-based evaluation.

### 10. Intentional Connection
> Remote ≠ isolated.

**Evidence:** Annual retreats (Help Scout, Automattic), FIKA/Donut pairing, optional social calls.

## Implementation by Company Size

### Small Team (< 50 people)
**Start with:** Basecamp model
- Daily Q&A check-ins
- Shape Up for projects
- Minimal meetings
- 6-week cycles
- Read: [basecamp-summary.md](./handbook-summaries/basecamp-summary.md)

### Medium Team (50-200 people)
**Start with:** Zapier + Help Scout hybrid
- Friday updates
- Tool purpose clarity
- Buddy system onboarding
- Annual retreats
- Read: [zapier-summary.md](./handbook-summaries/zapier-summary.md), [helpscout-summary.md](./handbook-summaries/helpscout-summary.md)

### Large Org (200+ people)
**Start with:** GitLab + Automattic model
- Handbook-first approach
- P2-style long-form async
- Maturity model assessment
- 24/7 timezone coverage
- Read: [gitlab-summary.md](./handbook-summaries/gitlab-summary.md), [automattic-summary.md](./handbook-summaries/automattic-summary.md)
